## O que é?
É a criação e padronização dos formulários de Issues do GitHub para o nosso projeto. Isso garantirá que toda nova tarefa, relato de erro ou proposta de estudo siga um padrão de informações.

## Como funciona?
Ao adicionar os arquivos Markdown formatados dentro da pasta oculta `.github/ISSUE_TEMPLATE/`, o próprio GitHub passa a reconhecê-los. Assim, sempre que alguém da equipe clicar em "New Issue", a plataforma abrirá um menu perguntando qual tipo de Issue a pessoa deseja criar (Rabbit, Feature, Bug ou Docs) e já carregará o texto base para ela preencher.

## Tópicos a serem executados:

- [ ] Criar a pasta `.github/ISSUE_TEMPLATE/` na raiz do projeto.
- [ ] Criar e preencher o arquivo `rabbit.md` (Para estudos de novos conceitos).
- [ ] Criar e preencher o arquivo `feature-request.md` (Para solicitação de novas funcionalidades).
- [ ] Criar e preencher o arquivo `docs-repair.md` (Para correções na documentação).
- [ ] Criar e preencher o arquivo `bug-report.md` (Para relato de erros).
- [ ] Fazer o commit atômico das alterações (ex: `chore: adiciona templates de issues`).
