Estudo (arquitetura): definição de tópicos de estudo

O que é arquitetura de Software?

-É a estratégia de organização do sistema, é o modo de direcionar e organizar o projeto definindo componentes, as conexões e as restrições quanto à documentação, stacks, framework e Api

Qual sua importância?

-É essencial para manter a coerência, a estabilidade e a manutenção do código. Principalmente em projetos em grupo que demandam esforço de várias pessoas, a arquitetura existe para organizar e manter uma linha contínua de desenvolvimento e evitar conflitos entre partes feitas por diferentes pessoas

Tópicos a serem estudados:


Documentação : Estudaremos como traduzir a lógica do sistema em artefatos visuais e textuais (como o Documento de Arquitetura), garantindo que a Wiki seja o "norte" para os desenvolvedores.


Stacks : Analisaremos as tecnologias escolhidas (Linguagem, Banco de Dados, Servidor) para garantir que sejam compatíveis entre si e que a equipe consiga configurar o ambiente via Docker sem conflitos.


Framework : Focaremos em entender os padrões impostos pelo framework escolhido (ex: Django, FastAPI ou Spring Boot), definindo como as camadas de Service, Controller e Repository devem ser organizadas.


Api : Definiremos a arquitetura de comunicação do sistema, estabelecendo padrões para as rotas, métodos HTTP e formatos de resposta (JSON), garantindo uma integração fluida entre o Frontend e o Backend.
