## O que é?
É a criação e implementação de um documento oficial (Guia Definitivo) que estabelece as regras de versionamento de código, arquitetura de branches, formatação de commits e fluxo de aprovação de Pull Requests para todo o time.

## Como funciona?
Ao padronizar o uso do Git (exigindo commits atômicos e uso do Conventional Commits) e a estrutura do GitHub (protegendo a branch `main` e centralizando o desenvolvimento na `dev`), a equipe ganha um ambiente de homologação seguro. Isso reduz drasticamente os merge conflicts, garante revisões de código consistentes através de templates em Pull Requests e mantém o histórico do projeto legível e escalável. O documento servirá tanto como *onboarding* para novos membros quanto como manual de consulta diária (Cheat Sheet).
